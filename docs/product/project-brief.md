# 项目简介

## 一句话说明

`easy-api` 是一个 OpenAI 兼容的智能网关，用来把“不稳定、经常变化的 OpenRouter 模型池”包装成一个更稳定、更容易接入的下游 API。

## 这个项目解决什么问题

直接使用 OpenRouter 有几个现实问题：

- 免费模型会变
- 同一个模型今天能用，明天不一定还好用
- 不同模型支持的模态不一样
- 调用方通常并不想自己维护回退链

`easy-api` 要做的，就是把这些复杂度尽量收敛到服务端。

调用方理想中的体验应该是：

- 继续按 OpenAI 风格发请求
- 不必自己手工挑模型
- 主模型挂了还能自动切下一个
- 图片请求必要时自动走预处理
- 想排查时能看到路由理由

## 目标用户

主要是这两类人：

- 想低成本使用 LLM、又不想天天追模型变化的开发者
- 已经有 OpenAI SDK 接入代码、希望尽量少改动就切到网关的人

## 产品定位

这个项目是：

- 智能 LLM 网关
- 免费模型优先路由层
- 多模态预处理编排层

这个项目不是：

- 聊天前端
- 单模型代理壳
- 计费平台
- 多租户后台系统

## 当前已经落地的范围

当前项目已经可以提供：

- `chat/completions`
- `responses`
- `models`
- `models/auto`
- `capabilities`
- `metrics`
- `route/debug`

以及这些服务端能力：

- catalog 同步
- 候选过滤和打分
- fallback
- 图片编排
- 流式输出
- 调试元数据
- NDJSON 遥测
- 进程内 metrics

另外，当前默认策略是：

- 免费模型优先
- 默认关闭免费模型自动降级到收费模型

## 当前产品原则

1. 客户端稳定性优先
2. 免费模型优先
3. 服务端决策优先
4. 明确规则优先于“看起来很聪明”
5. 可观测优先于黑盒自动化

## 当前边界

当前版本明确没有做这些事情：

- 客户端鉴权
- 账号体系
- 计费与额度
- 会话存储
- 持久化对话历史
- 完整覆盖 OpenAI 所有接口和字段
- 默认不开启免费转收费降级

## 使用时要有的预期

这个网关能显著减少“接入和维护模型”的麻烦，但它不能消灭上游模型池本身的不稳定性。

尤其是免费模型，始终会有这些现实情况：

- 可用性波动
- 指令服从性波动
- 延迟波动
- 模型列表变化

所以这个项目的目标从来不是“保证永远最强”，而是“在当前条件下，尽量给出一个更稳、更能用的选择”。
