## 核心研究问题

我们能否构建一个工具，自动发现 OpenRouter 上当前“完全免费”的模型，把它们接入使用，并自动选择其中相对表现最好的那个？

## 子议题

1. OpenRouter API 基础
预期获得的信息：
- 官方 API 使用方式
- 认证与请求格式
- 用于列出模型和调用 `chat/completions` 的相关 endpoint

2. 免费模型发现
预期获得的信息：
- OpenRouter 是否暴露模型定价元数据
- 是否能通过 API 可靠判断“完全免费”
- provider 级别定价、限制或临时可用性方面是否有注意事项

3. 自动选择最佳模型
预期获得的信息：
- OpenRouter 是否暴露 benchmark、排名、上下文、延迟或吞吐元数据
- routing 或 provider 偏好是否能提供帮助
- 在一个生产工具里，“最佳”现实中到底意味着什么

4. 产品可行性
预期获得的信息：
- 一个实用工具的架构形态
- 主要工程和产品风险
- 这个想法是否能直接成立，还是需要先收窄范围

## 综合方式

优先使用 OpenRouter 官方文档，然后总结：

- 哪些能力是官方直接支持的
- 哪些部分需要靠推断或自定义逻辑
- 推荐的 MVP 范围与风险控制方式
